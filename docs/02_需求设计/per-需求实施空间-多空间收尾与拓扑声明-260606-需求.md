## 需求描述

req cmq23elzh081b0a36b7726299（横跨主仓+plugin+su-oriel 三仓）收尾暴露规则缺口：需求级合并 `mergeRequirementWorktree()` 只覆盖 superproject 单空间，子仓实施分支与 gitlink 抬升游离在规则之外——主仓 merge 后 main 上测不到本批代码，4 个历史跨仓需求（913778/9dd2ae/718db0/1db492）的子仓收尾全靠对话内手动操作。归因为规则缺口而非执行违规："子仓独立 worktree"是 2026-06-03 执行期即兴补救，从未回流成规则；726299 批注记"留用户授权合并流程"指向一个不存在的流程。

本需求把收尾规则的本体从"仓"重构为"需求名下的实施空间集合"：dispatch 展开并 ensure 全部实施空间；子任务全部 done 后，全部空间各自合并回各自运行态记录的源分支，空间间版本关联按项目机器可读拓扑声明由 lib deterministic helper 同步（SU-CCB：gitlink 抬升），整体进入 merged 预览暂停态等人工 check；任一空间失败整体 escalated 并保留现场。kernel 措辞零拓扑词汇，可分发下游项目。

## 三、讨论与决策

用户三轮建模修正（原话见 verbatim）：规则量词不盯"仓"，盯"需求+任务"——凡为需求开的实施分支，无论哪个仓/层级，done 后全部合回源分支等人工 check；superproject 关联（su-oriel 关联其他项目版本/commit）是项目特有知识，作为追加表述或项目文档声明。

consult `job_9c671ea3764e`（main_codex，2026-06-06，rep_9973ea0dbd54）：

- **建模认可**，本体从"实施分支"精化为"**实施空间记录**"：最小记录含 `kind/repo/path/worktree_path/branch/target_branch/base_sha/status/last_error`——分支名不足以支撑部分失败恢复
- **否决纯 prose+LLM 路径**（用户与 Claude 的原始倾向）："拓扑声明若只是给 LLM 读的 prose，会再次退化成手工流程"——正是本缺口形成机制。拓扑声明必须机器可读版本化配置，runtime 只记录展开后的实例
- **否决在现有 merge 里补 submodule 扫描**：把 SU-CCB 拓扑硬塞 kernel/lib，仍无法表达 N 个 target_branch、部分失败、恢复
- **gitlink bump 归 lib**：deterministic association helper，LLM 只调用+解释异常；同步产生的 commit sha 必须记入运行态（否则无法解释"代码已进子仓但主仓不可测"）
- **边界方案**：ensure 在 dispatch 派工前展开全部空间，子仓 target_branch 在各自 repo 捕获不继承 superproject；partial merge 按 per-space status 恢复；reopen 仅 requirement 级 all-or-nothing；任一空间 merged 禁 discard；cleanup 复用 260606 删除语义
- **执行序**：先落 260606（cleanup 底座），本需求不并入、不依赖其改 merge
- **right-size**：5 子任务；analysis_depth_hint: sc-design（kernel/runtime/项目拓扑三层契约变化，非局部 bugfix）

用户在量级诚实重报（单子任务估计作废 → 5 子任务设计级，无更小够用版）后拍板：立项。

## 四、功能 / 范围

1. **ADR-0036 amend + kernel manifest 措辞**：收尾量词改为"该需求名下全部实施空间"；新增不变量"全部空间 merge 完成 + 项目声明的关联同步完成 + 预览空间自洽可测 → 整体 merged；任一失败 → 整体 escalated 保留现场"；dispatch 措辞改为展开并 ensure 全部实施空间
2. **项目拓扑声明契约**：机器可读版本化 yaml（最终路径/schema 名技术设计拍板，候选 `docs/.ccb/config/implementation-topology.yaml`），声明实施空间种类与空间间版本关联（SU-CCB：superproject+submodule → 子仓合并后 gitlink 抬升）；runtime 只记录按声明展开后的空间实例
3. **runtime schema 多空间化**：`docs/.ccb/worktrees/<req>.json` 从单记录扩为 N 空间记录（per-space status + aggregate status），兼容读旧单记录；Console projection 最小适配
4. **dispatch/materialize/ensure**：派工前按拓扑声明展开并 ensure 全部实施空间；子仓 `target_branch` 在各自 repo ensure 时捕获记录，不继承 superproject
5. **merge/reopen/discard/cleanup orchestrator**（含部分失败恢复与测试）：
   - merge：逐空间合回各自记录的 target_branch；partial 失败 → aggregate escalated，保留已 merged 空间事实，恢复只重试未成空间不重放
   - 关联同步：全部空间 merged 后执行，作为 post-merge recorded action，commit sha 与目标分支入运行态
   - reopen：仅 aggregate merged 可整体 merged→ready，全部 merged 空间校验 clean 后复用；partial escalated 不走 reopen 走 resume/人工
   - discard：仅全部空间 ready 时整体 discard；任一空间已 merged 即禁止（防已回流 target 的提交被静默丢弃）
   - cleanup：aggregate merged 后逐空间 cleanup；任一失败 aggregate escalated，成功空间保留 archived 事实、失败空间保留现场；删除语义复用 260606
6. **SU-CCB 项目拓扑声明落地 + gitlink association helper + 端到端验收**

## 五、业务规则

- kernel 零拓扑词汇（不写 submodule/gitlink/superproject），只写量词+不变量；拓扑知识归项目层机器声明——下游单仓项目零声明时规则自然退化为单空间合并
- 每个空间的 merge target 永远用该空间运行态记录的 `target_branch`，绝不 fallback、绝不跨空间继承（继承 ADR-0036 不变量 3 并推广到 N 空间）
- `merged` 仍是预览暂停态非交付终态（继承 260604 语义）；整体 merged 的定义收紧为"全部空间 merged + 关联同步完成 + 预览自洽"
- escalated 必须保留现场（worktree、分支、运行态不变）；关联同步产生的 commit 必须可追溯（sha 入运行态）
- LLM 在收尾流程中只调用 lib 与解释异常，不得凭自然语言自由执行 git 拓扑动作

## 六、边界 / 不做项

- targeted reopen 单个空间：不做（minimal-first）；reopen 只做 requirement 级 all-or-nothing
- 不改分支策略：per-需求×N 空间、子任务串行共享（ADR-0018/0036）不动
- 不并入 260606、不依赖它改 merge；执行序在其落地之后
- 不做全 lib Git 异常治理（260606 立项时已划界另立项）
- 不动 Console 业务写入（仅投影最小适配）
- 不处理 push/PR/远端同步——merged 预览态仍是纯本地语义，外发动作仍归用户授权

## Claude 解读

本需求把 260604 建立的"merged 预览暂停"语义从单 superproject worktree 推广到"需求名下全部实施空间"。本体=实施空间记录（非分支名、非仓），最小事实集 kind/repo/path/worktree_path/branch/target_branch/base_sha/status/last_error，支撑部分失败恢复。kernel 只写量词与不变量——"该需求名下全部实施空间完成 merge + 项目声明的空间间关联同步完成 + 预览空间自洽可测，才整体 merged；任一失败整体 escalated 保留现场"——零拓扑词汇，可分发下游项目。SU-CCB 的 superproject-submodule 拓扑下沉为项目层机器可读声明（路径/schema 名留技术设计拍板），gitlink 抬升由 lib deterministic association helper 执行并把 commit sha 记入运行态，LLM 仅负责调用与解释异常、不凭自然语言自由发挥。状态机安全语义：partial merge 按 per-space status 恢复（只重试失败空间，不重放已 merged 空间）；reopen 仅 requirement 级 all-or-nothing；任一空间已 merged 即禁止 discard；cleanup 在 aggregate merged 后逐空间执行、复用 260606 的删除语义底座。right-size：5 子任务（①ADR-0036 amend+kernel 措辞+拓扑配置契约 ②runtime schema 多空间化兼容旧单记录+projection 最小适配 ③dispatch/materialize/ensure 展开 N 空间 ④merge/reopen/discard/cleanup orchestrator 含部分失败恢复与测试 ⑤SU-CCB 拓扑声明+gitlink helper+端到端验收），走深度设计（consult hint: sc-design，三层契约变化非局部 bugfix）。执行序：先落 260606，本需求不并入、不依赖其改 merge。
