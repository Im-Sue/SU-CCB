# SP-B20 · 任务看板页 /tasks 重设计

## 1. 背景与范围

基于 SP-A10 路径 E（取消 Epic + Requirement 升格）+ SP-B15 拍板（子任务命名 + 6 极简状态 + 大状态独立指令），重设计任务看板页（`/tasks`）。

按 Console redesign 范围：UI / 交互 / 流程 / 按钮 ↔ 指令对应；不讨论指令内部逻辑。

## 2. 现状基线（Claude + codex 双扫描）

### 2.1 页面结构

|---|---|

| TasksBoardView.tsx | 398 |

### 2.2 当前布局（7 列 + 顶部 strip 们）

```
顶部 sticky:
  HealthPanel / UnstartedRequirementStrip / EpicSummaryStrip / TasksFilterBar

中部 (7 列):
  需求分析 → 技术设计 → 任务拆分 → 派工 → 执行实现 → 评审验收 → 归档收尾
```

### 2.3 现状关键

- 分组：`currentNode`（7 节点列）
- Epic 默认在顶部 strip，不在 7 列内
- 卡片字段 8-9 个
- 视图：Board / List
- 编辑只 4 个 operational 字段
- 任务列表无分页

## 3. codex round 8 协商结论

- 派工：`job_3fa6833c221f` / 回执：`rep_cf68ad2a25c9`
- analysis_depth_hint: medium / hint_confidence: high

### 3.1 必做（4 项）

| # | 内容 | 价值 | 复杂度 |
|---|---|---|---|
| 1 | 删 Epic 摘要 / Epic badge / 所属 Epic 列 → 改"所属需求" crumb | 符合两层模型 | 中 |
| 2 | 看板只展示子任务（不平铺 Requirement） | 定位清晰 | 中 |
| 3 | **7 列 → 4 列子任务执行列**；planning 三节点回需求详情页 | 减少概念倒置 | 中高 |
| 4 | "未立项需求"改名 / 移出（不再出现"立项"） | 避免旧心智 | 低 |

### 3.2 建议做 / 可选 / 不建议本轮

- **建议做**：按需求分组视图切换 / 子任务卡显示队列序号 + 等待中灰显 + 软停标记 / 筛选补充 / 已归档折叠或独立视图
- **可选**：保留保存视图 / 键盘导航 / 批量刷新 / 紧凑密度切换
- **不建议本轮**：拖拽改状态 / 自定义列 / 批量 cancel/defer

### 3.3 风险

| # | 风险 | 缓解 |
|---|---|---|
| R1 | Epic 消失，旧用户找不到总览 | 顶部 Requirement 总览 strip 替代 |
| R2 | 看板同时承载 planning + execution → 过密 | 必做 #3 把 planning 三节点搬走 |
| R3 | 跨需求汇总 vs 单需求队列导航不稳 | 看板 vs 详情职责分离（§4） |

## 4. 关键设计洞察

**任务看板 vs 需求详情职责分离**：

- **任务看板** = 跨需求子任务"操作台"（execution layer，后 4 节点）
- **需求详情** = 单需求"全景视图"（planning + queue，前 3 节点 + 子任务队列）

7 节点中：
- 前 3 节点 → Requirement 维度 → 在需求详情页
- 后 4 节点 → 子任务维度 → 看板 4 列

## 5. 用户拍板（2026-05-18）

用户接受全部 codex 推荐 + Claude 推荐：

| # | 问题 | 拍板 |
|---|---|---|
| 1 | 默认分组 | **执行阶段（4 列）** |
| 2 | "按 Requirement 分组"视图 | **保留**（视图切换其中一个）|
| 3 | 子任务列 | **4 列执行态**（不兼容 7 节点）|
| 4 | 队列展示 | **全部展示**（同需求内等待项灰显）|
| 5 | cancel/defer 按钮位置 | **只在需求详情页**（不进看板）|
| 6 | 已归档 | **折叠保留**（默认隐藏，可一键展开）|

## 7. 按钮 → 指令对应（已加入 SP-B92 mapping）

| 按钮 | 指令 | 存在性 |
|---|---|---|
| 视图切换 | URL `?view=...` | 💠 |
| 子任务卡片点击 | navigate `/tasks/:id` | 💠 |
| 卡片"所属需求" crumb | navigate `/requirements/:rid` | 💠 |
| Requirement 总览 strip 项点击 | navigate `/requirements/:rid` | 💠 |
| HealthPanel 修复漂移 | POST `.../status-repair` | 🔧 |
| 顶部 ⟳ 刷新 stale | POST `.../refresh-stale-projections` | 🔧 |
| 筛选 / 搜索 | URL 更新 | 💠 |
| 已归档展开 / 折叠 | UI 状态 | 💠 |

**注**：cancel / defer / resume 按钮**不在看板**（拍板 5）→ 进需求详情页；派工 / 评审 / 归档子任务也在子任务详情页操作（看板只是导航入口）。
