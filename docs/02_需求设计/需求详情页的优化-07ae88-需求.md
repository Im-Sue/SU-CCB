## Claude 解读

本需求聚焦需求详情页的「文档 / 产物阅读体验」整理与滚动修复，共四项前端改动（N1–N3 沿用既有分析结论作证据，N4 为 2026-05-25 本轮新增）：

- **N1 去冗余按钮**：「展开」与「全屏阅读」打开同一只读弹窗，功能重复。删「展开」，留「全屏阅读」；「编辑需求文档」不动。
- **N2 去重复描述块**：卡片上方 verbatim 原话块与下方 ≤220 字摘要块同源重复。删上方原话块，留下方摘要（≤220 字截断不变）。需求文件原话永久保留。
- **N3-a 编辑态滚动（P0）**：高度/overflow 约束错放在外层 `.CodeMirror`（overflow:hidden），真实滚动层 `.CodeMirror-scroll` 未被约束。把约束移到 `.CodeMirror-scroll`，并为 `.CodeMirror-fullscreen` 留例外。
- **N3-b 阅读态渲染**：「全屏阅读」改用 `MarkdownViewer` 渲染（含资产 URL rewrite 抽取、read 弹窗 scoped 溢出兜底、editError 仅 edit 显示）；编辑态续用 EasyMDE。
- **N4 阅读设计显示正文（本轮新增）**：详情页「阅读设计」抽屉现渲染 `buildDesignMarkdown` 的指针文本（"已生成文档：<路径>…仅提供产物索引和快速入口"），而非设计文档正文。改为读取并渲染正文。

**N4 重推边界（已核验、非锚定）**：happy path 为纯前端——详情页用已加载的 `documents` 索引把 `planDocPath` 解析成 documentId（path 轻量 normalize：trim、去开头 `./`、反斜杠转 `/`，不折叠大小写），复用既有 `GET /api/documents/:documentId` 读取链路 + `MarkdownViewer` + 抽共享的 `stripFrontmatter`，在抽屉渲染正文。**无需新增后端端点 / schema / 依赖**。N4 非「纯 JSX」：需接 detail store 的加载 / 错误 / 清理状态。fallback 覆盖：文档未索引、索引 stale、fetch 404 / 读失败、store 残留旧文档、加载中、空正文、frontmatter stripping。MarkdownViewer 横向溢出兜底与 N3-b 同批做。

**整体范围（依「范围非锚定」以 3 点+N4 完整内容重推）**：前端为主 + 复用既有 document 读取 API；不新增后端端点、不改 schema / 字段 / 存储、不加依赖。N1/N2/N3 的「纯前端」边界经本轮核验仍成立（是重推结论，非沿用旧边界）。N4 需补一次 technical_design pass 落实现设计。
## 歧义点

- **产品取向（N4）**：「阅读设计」应「抽屉内嵌正文阅读」还是「维持索引 + 跳转文档中心」？当前指针文本是先前有意设计。用户本轮原话「而不是具体的文档内容？这是不是设计和交互的不合理」明确指向「应显示正文」，故按「抽屉内嵌渲染正文」重推。此为推翻先前 index-only 设计的决定——若用户其实想「点击跳转文档中心」而非内嵌，可否决本结论。
- **是否统一产物阅读模式（N4）**：是否顺带把「阅读解读」（渲染 requirement 投影字段）与「阅读设计」（读文件正文）统一为同一组件 / 模式？经 Codex 协商（job_1166abcbf933）判断不顺带：AI 解读是 requirement 投影字段、技术设计是文件产物，性质不同；N4 保持最小，仅修设计抽屉，统一另开需求。
- **Claude 自决（低影响、非用户权利）**：N1 保留「全屏阅读」标签、N2 维持 220 字截断、N3 / N4 的具体技术实现路径（path normalize、fallback 分支、stripFrontmatter 抽取、溢出兜底）。
- **未命中**：schema / 隐私 / 合规 / 成本 / 外部服务 / 不可逆工程动作——N4 复用既有只读接口、无新数据契约；N1–N3 为纯 CSS/JSX。
