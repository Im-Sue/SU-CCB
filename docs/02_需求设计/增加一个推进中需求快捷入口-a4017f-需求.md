## 需求描述

对于绑定了slot的需求，右下角全局悬浮，然后点击可以展开有哪些需求，然后再点击对应需求之间跳转到对应需求详情页

## 原话（verbatim）

对于绑定了slot的需求，右下角全局悬浮，然后点击可以展开有哪些需求，然后再点击对应需求之间跳转到对应需求详情页

## 二、背景与目标

- 背景：SU-Oriel 多 slot 并行推进需求；用户要切到某个正被 agent 占用 slot 的需求详情页，当前得回需求列表再找，路径长。
- 目标：右下角全局悬浮入口，一键展开『当前绑定了 slot 的需求』，点击直达其详情页，缩短跨需求导航路径。
- 非目标：不做需求状态管理面板；不做 slot 绑定/释放/取消操作(那是 slots 页职责)。

## 四、功能 / 范围

范围内：
1. 全局悬浮入口(右下角)，各路由可见。
2. 点击展开『绑定 slot 的需求』列表(当前选中 project)。
3. 列表项显示需求 title + slot/state 角标；点击 navigate 到 /requirements/:id，跳后收起。
4. 空态(无绑定需求)：入口隐藏或显示空提示(设计阶段定，默认隐藏)。

范围外：跨 project 聚合；slot 绑定/释放/取消操作；按生命周期(planning/delivering)筛选；排队(queued)需求。

## 五、业务规则

- 『绑定 slot 的需求』= 当前 project 的 `SlotProjectionView.slots` 中 `requirement != null` 的全部条目，覆盖 bound/busy/unhealthy/recovering/draining 全部占用子态。
- 排除：main coordination lane(无 requirement)；`queued[]` 排队项(未占用 slot)。
- 同一需求占多个 slot 时按 requirementId 去重，列一条。
- 在需求详情页时，当前需求那条可高亮/置灰，但不从列表移除(FAB 价值是跨需求切换)。
- 数据只读，不回写。

## 六、边界 / 不做项

- 不改后端 API/schema/DB；只读消费现有 `fetchSlots`。
- 不新增运行时依赖。
- 不把 Toast 并进业务 dock；Toast 只做位置避让。
- 不为未落地的 ea509d 预建通用浮层框架(避免 scope anchoring 式 over-build)。

## 八、拆分预览

单一前端任务量级：新增悬浮入口组件 + 取数(组合 slots 投影) + 路由跳转 + 与 Toast 的定位避让；预计 1 个开发子任务(含样式与空态)。详细拆分在 task_breakdown 节点。

## 九、数据(草案)

无新增数据模型。消费现有 `SlotProjectionView.slots[].{ requirement: { id, title }, state, slotId }`。

## 十、接口(草案)

无新增接口。复用 `fetchSlots(projectId)`(GET /api/projects/:id/slots)。

## 十一、界面 / 页面布局

- 收起态：右下角悬浮按钮(FAB)，与 Toast 错位避让(Toast z-index 200，FAB 取不遮挡层级)。
- 展开态：上方弹出小面板，列出绑定 slot 的需求(title + slot/state 角标)，可滚动。
- 风格：沿用 CSS modules + design tokens；参考 Toast 的右下角定位与动画。

## 十二、交互 / 流程

1. 点 FAB → 展开列表(取当前 project 绑定 slot 的需求)。
2. 点某条 → navigate(/requirements/:id) → 收起面板。
3. 列表为空 → 入口隐藏(默认)或显示『暂无绑定 slot 的需求』。
4. 再点 FAB 或点外部 → 收起。

## 十三、风险

- 语义混淆：文案若用『推进中』会与 canonical `planning` 冲突 → 缓解：用『Slot 中需求 / 绑定 slot 的需求』。
- 陈旧/成本：全局轮询增加后台请求，不轮询又可能陈旧 → 设计阶段定刷新策略。
- 右下角拥挤：与 Toast / ea509d 入口遮挡 → 薄共享定位 + 避让；统一 dock 另案。
- 详情页重复：详情页已显示当前需求 slot 状态，FAB 部分重复 → 当前需求项降级展示，不隐藏整个入口。
