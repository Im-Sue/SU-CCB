## 需求描述

文档中心界面交互还是太混乱了， 更直观点。 不需要复杂的文档逻辑和汇总之类的

## 原话（verbatim）

文档中心界面交互还是太混乱了， 更直观点。 不需要复杂的文档逻辑和汇总之类的

## 二、背景与目标

> 一句话：把文档中心从"治理中心"简化回"找文档 → 打开阅读"的直观浏览器。

- **背景**：第一次重构（需求 `2318ee`，2026-05-29 交付，commit `96aaa49`）把文档中心从扁平列表改成**三栏治理中心**——中栏塞入文档覆盖汇总（需求 / 有设计 / 有任务 / 缺口）、健康度、未绑定三段聚合。实际使用后用户认为界面交互"还是太混乱"，治理聚合属"复杂的文档逻辑和汇总"，非高频所需。
- **目标**：
  1. 主流程收敛为单一心智："找文档 → 打开阅读"，去掉多入口 / 多聚合叠加。
  2. 移除中栏全部治理聚合（覆盖 / 缺口 / 健康度 / 未绑定）。
  3. 保留目录导航与 Markdown 阅读体验不退化。
  4. 档位（生效中 / 历史 / 归档）降为轻量筛选 / 小标记，不再作主分组。

## 三、讨论与决策

### 讨论：本轮是对"刚交付的第一次重构"的简化 / 部分回退

- 张力：治理中心方向是上一版**用户亲自拍板**的；本轮用户使用后改变诉求要求简化。
- **最终决策 + 理由**：按硬约束【范围非锚定】，不以上一版"治理中心" framing 约束本轮；上一版覆盖 / 健康聚合恰是本轮要移除项。上一版分析仅作历史证据，不作准入门槛。

### 讨论：目标形态（A / D / C / B）

- 候选：A 两栏目录树浏览器 / D 轻量平铺列表 / C 极简单列表 / B 最小改动（仅删中栏）。
- **最终决策 = A（用户 2026-06-01 拍板）**：左栏按 docs 目录折叠树 + 搜索，右栏 Markdown 阅读。保留目录结构作主导航，砍掉中栏聚合。
- 否决理由：D / C 弱化或取消目录结构，用户选择保留目录树；B 保留"目录×档位"双重分组，档位分组本身可能仍被视作"复杂"。

### 讨论：档位（生效中 / 历史 / 归档）

- **最终决策 = 降为可选筛选 / 小标记（用户拍板）**：不作主分组，保留为可选筛选项或文档项小标记，按需过滤归档 / 历史。

### 讨论：移除边界（前端 vs 后端）

- 经 slot2_codex 协商核验：后端 `deriveDocumentGovernance` 产出的 per-doc `governance` 字段（tier / parseError / requirementId）同时被 documents list route 与 indexer / document-map 消费。
- **最终决策**：本轮只简化**前端 UI 呈现**，**不删后端 governance 字段**（additive、跨消费方）。"不需要复杂逻辑和汇总"针对 UI，不等于删后端派生。

## 四、功能 / 范围

### 4.1 左栏 · 目录树导航 + 搜索

| 项 | 说明 |
|----|------|
| 做什么 | 按 docs 目录结构（00–06 + 99_归档）组织为可折叠树，列出各目录下文档；顶部保留搜索框（按标题 / 路径模糊匹配）|
| 谁用 | 人类 operator |
| 怎么触发 | 展开 / 折叠目录、输入搜索、点击文档项 |
| 预期结果 | 一眼看到目录结构，点击文档即在右栏阅读 |

### 4.2 右栏 · Markdown 阅读器（保留）

| 项 | 说明 |
|----|------|
| 做什么 | 复用现有 `MarkdownViewer` 渲染选中文档；保留标题 / 路径 / 类型徽标 + frontmatter 元数据可折叠 |
| 预期结果 | 阅读体验不退化 |

### 4.3 档位筛选 / 小标记

| 项 | 说明 |
|----|------|
| 做什么 | 档位（生效中 / 历史 / 归档）作为可选筛选项或文档项小标记，默认不分组 |
| 预期结果 | 需要时能过滤归档 / 历史，平时不干扰主流程 |

### 4.4 解析异常小标记（轻量保留）

| 项 | 说明 |
|----|------|
| 做什么 | 单个解析失败文档给一个轻量 ⚠️ 小标记（per-doc，非聚合健康度区）|
| 预期结果 | 坏文档有提示，但不再有独立"健康度"聚合分区 |

## 六、边界 / 不做项

> 负向验收（明确不做，防止"治理心智"残留 → 避免第三次返工）：

- ✗ 不展示文档覆盖汇总数字（需求 / 有设计 / 有任务 / 缺口）
- ✗ 不展示需求↔设计↔任务的缺口计算 / 覆盖卡
- ✗ 不展示健康度**聚合区**（parseError 列表）—— 仅保留 per-doc 小标记

- ✗ 不动后端 `deriveDocumentGovernance` / `document.routes` / indexer document-map（per-doc governance 字段保留）
- ✗ 无 DB migration、无新增重依赖（做减法 + 复用 `MarkdownViewer` / `Card` / `Badge` / `EmptyState`）

### 核心判断：这是对"刚交付的第一次重构"的简化 / 部分回退

第一次重构（需求 `2318ee`，commit `96aaa49`，2026-05-29 交付）把文档中心建成三栏**治理中心**——中栏覆盖汇总（需求/有设计/有任务/缺口）+ 健康度 + 未绑定三段聚合。本轮原话"**还是**太混乱""**不需要**复杂的文档逻辑和**汇总**"正指向那些聚合。方向：收敛回"找文档 → 打开阅读"的直观浏览器。⚠️ 按硬约束【范围非锚定】，不以上一版治理中心 framing 约束本轮——上一版聚合恰是本轮要移除项。

### 已拍板方向（用户 2026-06-01）

- **形态 = A 两栏目录树浏览器**：左 = docs 目录折叠树 + 搜索，右 = Markdown 阅读；中栏聚合全部移除。

### 范围边界（含 Codex 核验）

### 协商与 4 锚点反思

- **盲点**：anchoring 在"中栏=罪魁"、误判前端投影爆炸半径、漏看后端 governance 被 document-map 消费。
- **下一步**：方向已闭合 → 转入 technical_design 节点，设计 A 形态前端重写 + 投影精简 + 测试改写。
