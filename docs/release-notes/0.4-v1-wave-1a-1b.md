## Added

- Wave 1A archived E1/E2/E3，共 3 epic、18 task。

## Changed

- Archived spec lint 从单一 strict 行为调整为 current strict + legacy baseline 分层。
- Capability resolver 默认进入 primary-only 模式，旧 hint path 保留为可逆 fallback。

- Release 记录开始引用 `distribution-version-matrix.yaml` 作为 canonical baseline。

## Fixed

- 修复 archived legacy spec 对新 lint gate 的阻塞。
- 修复 manifest strict 之前缺少 refs-only、guard 与 capability 引用硬门槛的问题。
- 补齐 Console V1 settings、hook demo 与覆盖率基线，降低 V1 虚报风险。
- 在 E6-T0 后修正 `e4-t4-coverage-baseline-report.md` 的 archive spec_path/spec_hash 漂移。

## Deprecated

- `analysis_depth_hint` 主仓使用进入 disable/remove 路径；Wave 1A 已完成 inventory 与 primary-only disable。
- 旧 hint/sc fallback 仍保留代码，但默认不主动调用。
- plugin kernel snapshot 旧 source commit 仍可读，但发布时必须声明 drift。

## Removed

- 无。Wave 1A/1B 未删除旧 hint/sc 代码、Task.phase 字段或子模块内容。

## Security

- Manifest strict gate 增加 guard/capability 引用一致性，降低无声明执行风险。
- Hook adapter demo 严格 no-op/audit envelope，不执行 state mutation。
- 本批未引入生产依赖；coverage 仅增加 server dev dependency。
