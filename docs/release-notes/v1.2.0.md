# v1.2.0 Release Notes

## Version Matrix

- Main repo: `SU-CCB` v1.2.0
- Claude plugin: `su-ccb-claude-plugin` v1.2.0
- Codex skills: `su-ccb-codex-skills` v1.2.0 tag alignment

## Added

- 通知收件箱：SU-Oriel 增加通知收件箱视图，集中查看 CCB 运行时消息与处理线索（`cmq7mj42`）。
- 需求看板分组优化：需求视图按状态/分组更清晰地呈现推进队列（`cmq4szij` / `d91293`）。

## Changed

- 合并 gate 按需求隔离并收口：canonical merge、association 外层 gate、gitlink executor 内层 gate 复用三态 dirty classifier。
- 并行需求不再因彼此的合法 docs / 协调件 / 需求资产变更互相阻塞；TOLERATE 只放行，不会被 owner 合并 stage 或删除。
- root canonical 写操作加入仓库级锁，降低多空间 closeout 并发写 git 状态的竞争。

## Fixed

- 归档按钮门控修复：SU-Oriel 归档入口按当前任务状态正确启用/禁用（`a654c0`）。
- 通知消息修复：通知内容与状态同步更稳定，减少漏报/误报（`cmq4tn9`）。
- dirty gate 对未跟踪 state/config/schema/contract 与未声明机器产物保持 FOREIGN，避免无归属变更被放行。
