# CCB Console 系统架构文档

## 1. 目标

| 项 | 内容 |
|---|---|
| 目标 | 为本地多项目提供统一的项目、文档、任务与进度管理界面 |
| 核心原则 | 文件是真源，数据库是索引层，管理台是控制面 |
| V1 边界 | 不做通用工作流编排器，不做节点设计器，不深度绑定 `/ccb:su-*` 运行时 |

## 2. 总体架构

```text
本地项目目录
  -> 文件扫描层
  -> 文档解析层
  -> 任务归并层
  -> 索引数据库
  -> 本地 API
  -> 前端管理台
```

## 3. 分层设计

| 层 | 目录建议 | 职责 | 输入 | 输出 |
|---|---|---|---|---|
| 展示层 | `apps/ccb-console/web` | 页面、组件、状态管理 | API 响应 | 页面与交互 |
| API 层 | `apps/ccb-console/server/src/api` | 对前端提供接口 | 前端请求、领域服务 | JSON 接口 |
| 领域层 | `apps/ccb-console/server/src/modules` | 项目、文档、任务、需求、同步业务逻辑 | 解析结果、数据库数据 | 领域对象 |
| 索引层 | `apps/ccb-console/server/src/indexer` | 扫描文件、解析文档、归并任务 | 文件系统内容 | 索引记录 |
| 存储层 | `apps/ccb-console/server/src/db` | 持久化索引和运行记录 | 领域对象 | 数据库表 |
| 文件接入层 | `apps/ccb-console/server/src/fs` | 读写目录、监听变化 | 本地路径 | 文件内容和变更事件 |

## 4. 运行模式

| 模式 | 说明 |
|---|---|
| 本地单机模式 | 默认运行模式，服务直接读取用户机器上的项目目录 |
| 多项目模式 | 同一个管理台管理多个本地项目路径 |
| 增量索引模式 | 通过文件监听和重扫任务维护文档与任务索引 |
| 手动初始化模式 | 项目初始化由用户在 UI 里显式触发 |

## 5. 核心服务

| 服务 | 职责 | 备注 |
|---|---|---|
| `ProjectService` | 创建项目、更新项目信息、路径校验 | 项目入口 |
| `ProjectInitService` | 执行项目初始化、生成基础骨架 | V1 只需支持基础动作 |
| `DocumentScanService` | 扫描 docs 与 `.ccb` 目录 | 面向文件系统 |
| `DocumentParseService` | 解析 `spec / plan / task` 文档 | 解析 frontmatter 与正文摘要 |
| `TaskDeriveService` | 从三类文档归并任务视图 | V1 关键服务 |
| `RequirementService` | 创建需求并生成文档骨架 | 新工作入口 |
| `SyncJobService` | 管理 `init / scan / parse / reconcile` 记录 | 可观测性核心 |
| `FileWatcherService` | 监听文档变化并触发增量更新 | 可后置为 Phase 2 |

## 6. API 资源边界

| 资源 | 典型接口 |
|---|---|
| `projects` | 列表、创建、详情、更新、初始化、重扫 |
| `documents` | 列表、详情、阅读、按项目筛选 |
| `tasks` | 列表、详情、更新、绑定文档 |
| `requirements` | 创建、列表、详情 |
| `sync-jobs` | 列表、详情、重试 |

## 7. 前后端职责边界

| 问题 | 前端负责 | 后端负责 |
|---|---|---|
| 页面布局 | 是 | 否 |
| 数据校验提示 | 基础校验 | 最终校验 |
| 文档解析 | 否 | 是 |
| 任务归并 | 否 | 是 |
| 文件读写 | 否 | 是 |
| 运行记录 | 展示 | 记录与查询 |

## 8. 关键架构约束

| 约束 | 说明 |
|---|---|
| 文件真源不可反转 | 文档依然是业务事实来源，数据库不替代文档 |
| 索引可重建 | 数据库中的文档索引和任务派生结果必须支持重建 |
| 本地优先 | 不依赖远程服务才能完成基础管理功能 |
| 轻耦合命令接入 | `/ccb:su-*` 后续接入时不侵入当前主架构 |
| 单人优先，多人预留 | UI 可先按单人实现，数据模型预留 owner/assignee/reviewer |

## 9. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 文档格式不统一 | 解析失败、任务归并不准 | 增加解析状态和人工校正能力 |
| 任务归并规则不稳定 | 看板结果不可信 | 优先支持显式标识，保留人工绑定 |
| 文件监听噪声 | 增量更新不稳定 | V1 先保留手动重扫兜底 |
| 命令升级 | 后续接口变化 | 命令执行记录独立建模，避免侵入任务主模型 |
