---

> 📌 **怎么用这份模板**
>
> 这是模块的"**最终态说明书**",给维护者和对接方看:讲清"它**现在是什么**、对外提供什么、怎么用、有哪些规矩"——**不是**讲怎么一步步建(那是《技术设计》)。
>
> **一份规格 = 一个独立模块。** 命名带上它所属的部分作前缀:
> - 系统是**单体 / 单项目** → 直接 `X模块规格.md`。
> - 系统**分独立项目**(前端 / 后端 / 后台管理 各是独立工程,见《架构》)→ 模块按所属部分命名:`前端-X模块规格.md` / `后端-X模块规格.md` / `后台管理-X模块规格.md`。**它们各自是独立模块**(可能同域、会互相调用),**不是**一个模块的几层;谁跟谁怎么连由《架构》串。
>
> **按复杂度填**:
> - **简单模块**:填【一 概述、二 对外接口、四 关键设计点、五 业务规则、九 FAQ】。
> - **复杂模块**(有状态 / 并发 / 跨模块对接):补全【三 数据与状态、六 对接与扩展、七 失败处理、八 并发与一致性】,多上依赖图 / 状态图。
>
> **不同部分的「对外接口 / 能力」含义**:前端 = 页面 / 组件 / 路由;后端 = API / 服务 / 对外函数;后台管理 = 管理页面 / 操作。
>
> 把 `[方括号占位]` 换成你的内容,用不上的段删掉。
