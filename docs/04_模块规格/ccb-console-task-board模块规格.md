# 任务看板模块文档

## 1. 模块目标

| 项 | 内容 |
|---|---|
| 模块名 | 任务看板 |
| 目标 | 将 `spec / plan / task` 归并为统一任务视图并展示推进状态 |
| V1 价值 | 从“文档集合”提升到“可管理的工作对象” |

## 2. 功能范围

| 功能 | V1 状态 |
|---|---|
| 看板视图 | 必做 |
| 列表视图 | 可选 |
| 任务详情 | 必做 |
| 修改状态/优先级/负责人 | 必做 |
| 文档绑定与跳转 | 必做 |
| 自动复杂流程推进 | 不做 |

## 3. 页面与组件

| 页面/组件 | 作用 |
|---|---|
| 任务看板页 | 看板主体 |
| 任务详情页 | 任务工作页 |
| `TaskBoard` | 看板容器 |
| `TaskCard` | 任务卡片 |
| `TaskDetailPanel` | 详情主组件 |
| `LinkedDocsPanel` | 关联文档关系 |
| `TaskPhaseBadge` | 阶段标签 |

## 4. 输入输出

| 类型 | 内容 |
|---|---|
| 输入 | 文档解析结果、任务归并规则 |
| 输出 | `Task` 视图、看板列、详情结构化信息 |

## 5. 依赖

| 依赖模块 | 原因 |
|---|---|
| 文档模块 | 任务来源于文档 |
| 同步模块 | 文档变化后需要重新归并 |
