### 目标

让经 su-flow / su-dispatch 流程发起的协商/派工自动用 resolver 选同组对端;跨组给软提示。按决策(a)不追求拦截手敲命令(见边界)。

### 范围(plugin skill helper / command-builder + 文档接线,含测试)

- [NEW] skill 级 helper / command-builder:封装'锚定自身 → 调 PR1 resolver → 产出 ccb ask 目标'。
- [MODIFY] su-flow / su-dispatch 发起协商/派工处:未显式 target 时先调 helper 拿对端;ambiguous/no_peer → 要求显式 target,绝不猜。
- 跨组软提示:helper 在目标跨组时产出 warning 或要求 reason(流程内自检),不静默;不阻断合法跨组。

### 验收

- 经流程发起、未显式 target → 自动得同组对端。
- 跨组 → warning / 要求 reason;合法跨组仍可完成。
- 显式 target 旧路径不变。

- 不拦截手敲 Bash 'ccb ask ...':按决策(a),真正强制力/拦截留给 PR4 的上游 runtime contract,不在本仓做 hook。
