## 二、任务分解

### 任务概述

真 tmux 端到端集成断言 + 真浏览器人工验收清单(P2 欠账并入)。本切片是需求验收门:自动部分代码化,人工部分产出清单交用户。

### 任务分解

1. ws.spec 真 tmux 集成(隔离 socket,参照既有真 tmux 测试模式)五断言:
   a. 连接后 pane 输出 N 行→scrollback 含全部 N 行(运行期历史核心);
   b. 输出超 scrollback cap→尾部与 tmux 末尾一致、最老行挤出;
   c. hidden→输出→visible:ring 内无缝补发 / 超 ring reset,无重复无丢失;
   d. pipe 槽位被占(预先 pipe-pane 模拟 anchor)→降级模式工作+状态可见,释放后重试升级;
   e. server 重启(recorder 重建)→先 off 再开,流恢复。
2. 真浏览器人工验收清单(markdown,交用户执行):滚轮连续回看连接前+连接后历史(核心体验);claude/codex CLI 真实 TUI 场景观感(alt-screen 语义);06c62eb host 外滚/双底部贴底回归;PageUp/触控历史入口检查(可访问性)。
3. 回执措辞(诚实边界):自动验证通过;真浏览器清单已产出未执行,待用户验收。

### 验收标准

- 五断言在真 tmux 下稳定通过(容忍 CI 抖动重试);全仓绿。
- 人工清单文档落地,含逐项操作步骤与预期。

### 边界 / 不做项

不补新功能代码(发现 bug 回 pr1-pr3 修);不动 anchor-terminal。
