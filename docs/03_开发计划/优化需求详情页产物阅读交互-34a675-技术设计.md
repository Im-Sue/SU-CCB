## 二、方案与架构

```
需求详情页 RequirementDetailPage
  ├─ 产物卡片「AI 解析」 ──阅读──┐
  │                              ├─► <Modal size="reader">   (替代 DetailDrawer)
  ├─ 产物卡片「技术设计」 ─阅读──┘
  │
  ├─ AI 解析弹窗内容: RequirementMarkdownState
  │     open(ai) → GET /api/projects/:p/requirements/:r/markdown
  │             → {path, content(body-only)} → MarkdownViewer + rewriteRequirementAssetUrls
  │
  └─ 技术设计弹窗内容: 复用现有 DesignDocumentState
        open(design) → planDocPath → documents 投影 → fetchDocumentDetail → stripFrontmatter

后端 (只读):
  GET .../requirements/:id/markdown
     └─ service loadRequirementMarkdownBody(prisma, projectId, requirementId)
            ├─ 校验 requirement ∈ project (同 loadRequirementMdHash)
            ├─ findRequirementMarkdown() 读盘
            └─ extractMarkdownBody() 去 frontmatter
```

| 关键原则 | 说明 |
|----------|------|
| 后端真源读盘 | AI 完整 md 经 findRequirementMarkdown 直接读盘，绕开 documents 投影的 stale / 未索引风险 |
| 复用而非重写 | 技术设计阅读保留现有 DesignDocumentState 取文状态机，仅换容器 |
| a11y 内聚基础组件 | focus 管理补进共享 Modal，而非另起 ReaderModal 复制 overlay/ESC |
| 只读不写 | 新端点只读盘返回，不碰 DB / schema |

**与现有系统的关系 / 边界**：

| 涉及模块 | 本设计如何动它 | 保留 / 不动什么 |
|----------|----------------|------------------|
| `components/ui/Modal.tsx` | 增强：初始聚焦关闭按钮、focus trap、关闭恢复焦点、新增 `reader` size、可选 `initialFocus` prop | 现有 sm/md/lg/xl、ESC / 遮罩关闭语义不变 |
| `RequirementDetailPage.tsx` | 两处 DetailDrawer→Modal；新增 AI 取文 state | 全屏阅读、拆分草案、子任务、slot 区不动 |
| `requirement.routes.ts` / `requirement-edit.service.ts` | 新增只读 route + service 函数；导出 `extractMarkdownBody` | 编辑 / reindex / hash 既有逻辑不动 |
| `DetailDrawer.tsx`(task-detail-v2) | 仅停用本页两处调用 | 组件本体保留（他页可能仍用，呼应"任务详情页可能重做"，不在此动它） |

## 三、关键决策与取舍

- **取文数据源**：选「独立只读端点 + service 封装读盘」；否决「塞进 detail 响应」(detail 被 10s 轮询，重复传全文、耦合阅读与详情加载)、否决「detail 暴露 mdPath + 走 documents 投影 fetchDocumentDetail」(依赖索引，正是要规避的 stale / 未索引风险，Codex 上轮已指出)。
- **后端分层**：读盘逻辑落 service `loadRequirementMarkdownBody(prisma, projectId, requirementId)`，沿用 `loadRequirementMdHash` 的「先校验 requirement∈project 再 findRequirementMarkdown」模式；route 只做 HTTP 状态映射。`extractMarkdownBody` 由私有改为导出复用，不复制正则。
- **a11y 容器**：选「增强共享 Modal」(focus 管理本应在基础弹窗，惠及编辑 / 确认等所有弹窗)；否决「专用 ReaderModal」(复制 overlay/ESC，且留其它 Modal 仍缺 a11y)。
- **初始焦点**：默认聚焦关闭按钮 / dialog，不假设首个可聚焦元素是编辑器(规避 RequirementMarkdownEditor/EasyMDE fullscreen 接管视口)；留 `initialFocus` prop 供后续定制。
- **内容范围**：body-only(去 frontmatter)，与文档中心 `stripFrontmatter(documentDetail.content)` 一致。
- **打开期刷新**：AI 弹窗打开期间若 `requirement.mdHash`(详情轮询已带)变化则重拉，避免外部改动后看旧内容；hash 不变不重复拉，避免阅读时内容跳动。

## 四、核心流程 / 逻辑

```
[AI 解析阅读]
点「阅读解读」→ setArtifactModal("ai")
  → effect(open(ai) || mdHash 变): GET /requirements/:id/markdown
     → loading → ready{content} | empty | error | not-found(404)
     → MarkdownViewer(rewriteRequirementAssetUrls(content, projectId))
关闭(ESC / 遮罩 / 按钮) → 焦点恢复到触发按钮

[技术设计阅读]
点「阅读设计」→ setArtifactModal("design")
  → 复用现有 design fetch（effect 触发条件由 drawer==="design" 改为 artifactModal==="design"）
  → 现有 idle/loading/ready/not-indexed/stale/not-found/error/empty 态不变
```

| 处理规则 | 说明 |
|----------|------|
| 迟到响应丢弃 | AI 取文沿用现有 requestId/seq 模式（同 design），避免快速开关错配内容 |
| 打开期重拉 | 仅当 mdHash 变化触发，避免每次 10s 轮询都刷新内容 |
| 焦点回环 | Modal 内 Tab/Shift+Tab 循环，关闭恢复 previousFocus |

## 十、迁移影响与风险

- **受影响**：共享 `Modal` 的所有使用方(编辑 / 确认 / 批量 / 解绑弹窗)；`RequirementDetailPage` 两处阅读；新增一个只读端点。
- **打法**：Modal 增强保持向后兼容(只加能力，不改既有 size / 关闭语义)；先补 Modal 单测与编辑弹窗回归，再切两处阅读；后端端点纯新增。
- **回滚 / 恢复**：纯增量，`git revert` 即可；无数据迁移、无状态变更。

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Modal focus 增强影响编辑弹窗(EasyMDE) | 中 | 编辑体验回退 | 默认聚焦关闭按钮不抢编辑器；编辑弹窗回归测试 |
| 打开期 md 被外部改 | 低 | 看到旧内容 | mdHash 变化时重拉 |
| 图片重写能力边界 | 低 | 非标准图片链接(纯链接 / HTML img / `../assets` / 引用式)不渲染 | 既有边界，列为不做项，不在本需求修 |
| 端点路径越权 | 低 | 读到非目标文件 | findRequirementMarkdown 限定 requirement 目录 + 校验 requirement∈project |
