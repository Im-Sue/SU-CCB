# web:数据层类型分层 + 客户端治理派生

## 目标
前端类型分层 + 把 list `governance` 派生成左栏分组、覆盖度、健康度数据,供 pr4 UI 消费。

## 范围
- `[MODIFY] web/src/types/document.ts`:拆 `DocumentBaseView` 公共形;`DocumentView = DocumentBaseView + governance`(list);`DocumentDetailView = DocumentBaseView + { frontmatter, content }`(**不带 governance**)。
- `[MODIFY] console-api.ts`:`fetchDocuments` 透传 `governance`(无新请求);`fetchDocumentDetail` 不变。
- `[NEW] web/src/lib/document-governance-projection.ts`:从 `documents[]` 派生 ① 按目录 + tier 分组 ② 覆盖度聚合(按 `requirementId` 判 `technical_design` / `dev_task` 是否存在 → 有设计/有任务/缺口)③ 健康度(`parseError`)+ 未绑定(`requirementId=null`)分组;mirror `node-board-config` 的 config-driven 派生风格。

## 验收
- projection 单测:有设计 / 有任务 / 缺口 / 未绑定 / `parseError` 各场景分组与计数正确;**覆盖度仅判"文档存在",不掺交付状态**。
- 类型分层不波及 detail:detail mocks / preview drawer / reconcile reader 无需 `governance`,`tsc` 绿。
- `pnpm --filter ccb-console-web test` + `tsc --noEmit` 全绿。

## 边界

- 不引树/图库。
