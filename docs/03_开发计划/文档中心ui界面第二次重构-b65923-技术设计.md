## 二、方案与架构

```
documents (project-store, server 派生 per-doc governance)
        │
        ▼
projectDocumentBrowser(documents)   ← 改名后的轻量投影
        │  仅产出 groups: { directory, documents }[]（按 full parent dir 分组、排序）
        ▼
  keyword 过滤(title/path) + tier 过滤(可选)
        │
        ├──────────────┐
        ▼              ▼
 左栏 navPane      右栏 readerPane
 目录折叠树         MarkdownViewer（不动）
 +搜索+档位筛选      点击文档 → /documents/:id
 +per-doc 异常小标记
```

| 关键原则 | 说明 |
|----------|------|
| 投影只分组不聚合 | 删除 coverage / summary / parseErrors / unbound，只保留按目录分组 |
| full parent dir 兜底 | 分组 key = 文档完整父目录（`docs/.ccb/state` 等多级目录不丢），MVP 不做递归树组件 |
| 档位筛选为主、标记克制 | 顶部轻量 tier 筛选；文档项仅给 `历史/归档` 小标记，`生效中`（默认）不贴标 |
| 后端零改动 | per-doc `governance` 字段（tier / parseError / requirementId）保留，仅前端不再消费聚合 |

**与现有系统的关系 / 边界**：

| 涉及模块 | 本设计如何动它 | 保留 / 不动什么 |
|----------|----------------|------------------|
| `lib/document-governance-projection.ts` | **改名** `document-browser-projection.ts`；`projectDocumentGovernance` → `projectDocumentBrowser`；只产 `groups`（按 dir）| 删 coverage/summary/parseErrors/unbound；不留兼容层 |
| `pages/documents/DocumentsPage.tsx` | 两栏重写，删 `governancePane` + `coverageTarget` | 右栏 reader / MarkdownViewer / detail 加载逻辑保留 |
| `pages/documents/DocumentsPage.module.css` | grid 三栏 → 两栏；删 gov/coverage/health 样式 | navPane / reader 样式保留并微调 |
| 后端 `deriveDocumentGovernance` / document-map | **不动** | per-doc governance 字段保留（被 list route + document-map 消费）|
| `tests/__snapshots__/e12-acceptance-snapshots.spec.tsx.snap` | 重生成受影响快照 | 其余快照不动 |

## 三、关键决策与取舍

> Claude/Codex 协商（job_e17ea725eeec）结论落点。

- **投影精简方式**：选 **O2 改名** `document-browser-projection.ts` / `projectDocumentBrowser`，**不留旧 export 兼容层**；否决 O1（原地改名只删字段——残留治理心智、旧测试存活）、O3（内联进页面 `useMemo`——目录规则绑死页面、难单测）。理由：消费点仅 3 处、改名切断「治理中心」代码心智、实际改动面低。
- **目录分组粒度**：**full parent directory 分组**（保持现 `directoryOf` 的 `lastIndexOf("/")` 语义），MVP **不做任意递归树组件**。理由：`docs/` 实际有多级目录（`docs/.ccb/state`、`docs/release-notes`、`docs/debug_img` 等），必须按完整父目录兜底**不丢文档**；但无需递归树的复杂度。
- **档位处理**：**筛选为主、标记克制**——顶部一个轻量 tier 筛选；文档项**仅给 `历史/归档` 小标记，`生效中` 不贴标**。理由：全部贴标会重新制造噪音，只标非默认档位更直观（Codex）。
- **parseError**：保留 **per-doc 异常小标记**，但**不叫健康度、不计数、不聚合、不做大红 badge、不新增排序入口**。理由：坏文档提示直观有用，但须克制以免「健康度面板残留」。
- **AI 自决 vs 用户授权**：必问 12 类（依赖 / schema / 公共 API / migration / 成本 / 合规 / 隐私）**全部不命中**——纯前端做减法、复用现有组件、后端零改动；投影改名属 web 内部 helper 非公共契约（Codex 确认）。方向已由用户拍板，故实现细节属 **AI 自决**，无新增用户拍板项。

## 四、核心流程 / 逻辑

```
进入文档中心
  → documents 列表(已含 per-doc governance)
  → projectDocumentBrowser(): 按 full parent dir 分组 + 排序
  → 应用过滤: keyword(title/path 模糊) ∧ tier(全部=不过滤)
  → 渲染左栏目录折叠树(组内文档 + kind 徽标 + 历史/归档标记 + parseError ⚠)
  → 点击文档 → navigate(/documents/:id)
  → detail-store 加载 → 右栏 MarkdownViewer(stripFrontmatter)
```

| 处理规则 | 说明 |
|----------|------|
| 分组 key | 文档 full parent directory（`directoryOf(path)`）|
| 组内排序 | 按 `path` 升序（沿用现有）|
| 组间排序 | 按 directory 名升序（00→99，取代原 tier×dir 排序）|
| 折叠态 | 页面 local state，按 directory key 记忆 |
| tier 筛选 | 值 = 全部 / 生效中 / 历史 / 归档；「全部」不过滤 |
| 空态 | 空搜索 / 空筛选结果 / 空目录 → placeholder 文案 |

## 十、迁移影响与风险

- **受影响**：前端 documents 页 + 投影 lib + 2 个测试 + 1 个快照；后端、DB、API 零影响
- **打法**：投影改名 → 页面两栏重写 → 测试改写补负向断言 → 快照重生成，单 PR 内闭环；改名后先 `tsc` 扫死引用
- **回滚 / 恢复**：纯前端、无数据 / schema 变更 → `git revert` 即可，零迁移风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 第三次返工（仍没命中"直观"）| 中 | 高 | 已用 4 形态让用户拍板 A + 负向断言锁死「不回聚合」|
| 多级 / 嵌套目录丢文档 | 低 | 中 | full parent dir 分组兜底 + 单测覆盖 `docs/.ccb/state` |
| parseError 标记过重像健康度残留 | 中 | 低 | 小标记非大 badge，不计数 / 不聚合 / 不排序 |
| 删投影字段引入死引用 / 类型错 | 低 | 中 | 改名 + 删字段后 `tsc` + 全量单测把关 |
