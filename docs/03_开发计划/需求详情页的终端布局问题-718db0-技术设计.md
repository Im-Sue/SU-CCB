## 一、设计概述

| 项 | 说明 |
|----|------|
| 名称 | su-oriel slot 终端可滚动性修复 |

## 二、方案与架构

xterm@6.0.0（已核验 `node_modules/@xterm/xterm`）默认 **DOM renderer**（仅 Unicode11 + WebLinks addon，无 WebGL/Canvas），相关 DOM 结构：

```
.terminalHost            ← 我们的 div（固定高，flex:1, min-height:0）
  .xterm                 ← height:100%（现状）→ 改 height:auto; min-height:100%
    .xterm-scrollable-element   ← xterm6 真正的滚动层（VS Code SmoothScrollableElement）
      > .scrollbar              ← 内置滚动条 UI（xterm6 在此，非 .xterm-viewport）
      .xterm-screen            ← 当前屏 = rows × cellHeight（DOM renderer 渲 rows 个 row-div）
    .xterm-viewport      ← xterm5 旧滚动层，xterm6 下基本空壳
```

根因：左右 split → pane 变高 → `frame.rows` 近翻倍 → `terminal.resize` 把 xterm 撑成 rows×cellHeight 高于 `.terminalHost`；`overflow-y:hidden` 裁掉底部，xterm 内置滚动只滚 scrollback、不滚“当前屏溢出 host”，故看不到底部且无可达外层滚动条。

| 关键原则 | 说明 |
|----------|------|
| host 单一纵向滚动 | `.terminalHost{overflow-y:auto}` + `.xterm{height:auto}`，DOM renderer 把整屏 row-div 全渲进 DOM，host 滚动即可物理看全当前屏 |
| 收敛 xterm 内滚 | xterm6 滚动在 `.xterm-scrollable-element`，仅隐藏 `.xterm-viewport` 无效；须收敛 scrollable-element 的内置条，避免与 host 双条/抢滚轮 |
| 贴底跟随移到 host | 跟随基准从 xterm buffer 改为 host 的 `scrollTop/scrollHeight` |

**与现有系统的关系 / 边界**：

| 涉及模块 | 本设计如何动它 | 保留 / 不动什么 |
|----------|----------------|------------------|
| `SlotTerminalSurface.module.css` | 改 host/xterm/内置滚动条的纵向滚动策略 | 横向 `overflow-x:auto`、深色皮肤、padding 不动 |
| `SlotTerminalFrameRenderer.ts` | 构造接收 host 元素，贴底跟随改 host 级 | 帧合批/`resize`/full-frame-clear/去尾换行逻辑不动 |
| `SlotTerminalSurface.tsx` | 把 `containerRef.current` 传给 renderer | 复制/粘贴/右键/可见性 effect 不动 |
| `useXtermTerminal.ts` | `scrollback` 取舍（见 D3） | 主题/字体/addon/`term.open` 不动 |
| `MainTerminalPanel`（复用同 surface） | 不改代码，自动受益 | — |
| 后端 / WS / `EmbeddedTerminal` | 不碰 | WS `READ_ONLY`、tmux 主权、FitAddon 路径全保留 |

## 三、关键决策与取舍（Claude×Codex 协商结论 · job_5f55c90041eb）

- **D1 方向 A（host 外层纵向滚动）**：选 A，因为最贴合用户「缺外部滚动条/看不到底部」且唯一能「看全」；否决 B（fit-to-height：不缩字放不下、且违背“前端不 resize 后端”契约，失真）、C（前端 resize 后端 pane：WS `READ_ONLY` 硬拒、与 ccb 原生 sidebar 共享冲突）、D（crop-to-bottom：丢 pane 上半，留作用户可否决回退）。
- **D2 滚动层是 `.xterm-scrollable-element`，非 `.xterm-viewport`**：Codex 纠正、已核验 `xterm.css`（行 227+ `.xterm-scrollable-element > .scrollbar`，dist 引用 scrollable-element 20×∶viewport 4×）。现状 CSS 注释基于 xterm5 旧模型（`.xterm-viewport` 主权），升级到 xterm6 后已失真——本设计据此改正选择器目标。
- **D3 保留历史（用户拍板·v1.1）+ 机制 Opt-1a'（v1.2 定稿·协商 job_866cf3d43c0f）**：**保留 `scrollback`**（用户：上翻历史是核心；v1.0 `scrollback:0` 被否决）。两个滚动空间（当前屏溢出 / xterm 历史）须共存。**定稿 Opt-1a'**：host 是**唯一可见纵向条**、**隐藏 xterm 原生 `.scrollbar.vertical`**（它绝对定位在 `.xterm` 内、host 外滚时移出视口、不可靠，故不作历史入口）；历史改由 **host capture-phase wheel 仲裁**进入——上滚先把 host 滚到当前屏顶、再 `terminal.scrollLines(-n)` 入历史；下滚先 `scrollLines(+n)` 回 live、再 host 滚到底。两条并存（Opt-1b）仅留作快速回退，不作正式方案。用户已授权滚动条机制自由调整，Opt-1a' 在授权内。
- **D4 跟随＝双底部、且在 resize 前测**：跟随条件改 `historyAtBottom && hostAtBottom`（Codex：仅测 host 不够——rows 变高且历史不在底时不应硬贴）；**必须在 `terminal.resize()` 之前**测（resize 后 scrollHeight 变会误判离底）；仅双底部成立才写后条件 `terminal.scrollToBottom()` + `host.scrollTop=scrollHeight`，二者均不无条件执行（否则破坏保留的历史位置）。

**4 锚点反思**：
- *我同意*：xterm6 滚动层是 `.xterm-scrollable-element`、原 viewport-only 选择器是 xterm5 残留（已核 xterm.css）；只隐藏滚动条而留 scrollback 是“藏问题”，置 0 才是干净修法。
- *我保留*：A1（host 单一滚动）优于 A2（host 滚当前屏+xterm 滚历史共存）——A2 滚轮归属不稳，Codex 同此判断；DOM renderer 使 host 滚动能物理看全当前屏（Codex 确认）。
- *我的盲点*：① 误用 xterm5 DOM 结构（viewport 当滚动层）；② 低估保留 scrollback 会渗入 wheel/选区/follow；③ 把“砍 Web 内历史”当纯 CSS 细节，实为需用户确认的能力变更；④ 漏了移动端 `.terminalColumn height:auto` 这一回归点。
- *接下来*：本设计落库→向用户确认 D3→进 task_breakdown（1 个 web-UI 全栈子任务，Claude 亲自 pickup）；浏览器级验证为强制项（jsdom 测不到 scroll/wheel）。

## 四、核心流程 / 逻辑

> **v1.2 定稿 Opt-1a'（下方 v1.0 示意图已被超越）**：跟随＝双底部 `historyAtBottom && hostAtBottom` 且**在 `resize()` 之前**测，仅双底部才条件 `scrollToBottom()` + `host.scrollTop=scrollHeight`；历史经 **host capture-phase wheel 仲裁**进入（到顶继续上滚 → `scrollLines(-n)`；下滚先回 live 再滚 host 到底），隐藏 xterm 原生 `.scrollbar.vertical`，保留 `scrollback`。

```
后端帧 frame{cols,rows,data,initial}
   │
   ▼
renderer.flush()
   ├─ sizeKey 变 → terminal.resize(cols, rows)        // 不变
   ├─ wasAtBottom = host.scrollTop+clientHeight >= host.scrollHeight-ε   // 新：基准改 host
   ├─ payload = initial ? data : (\x1b[H\x1b[2J + data)  // 不变
   └─ terminal.write(payload, () => {
         if (wasAtBottom) host.scrollTop = host.scrollHeight;  // 新：贴底改 host
         terminal.scrollToBottom();                            // 保留：维持 buffer
      })
```

| 处理规则 | 说明 |
|----------|------|
| 双底部才跟随 | `historyAtBottom && hostAtBottom`（resize 前测）才贴底；否则不弹走用户 |
| 单一可见条 | host 条唯一可见；隐藏 xterm 原生 `.scrollbar.vertical`，历史走滚轮/PageUp |
| 滚轮仲裁 | host capture-phase wheel：到顶继续上滚 → `scrollLines(-n)` 入历史；下滚先回 live 再滚 host 到底；尊重 mouse-tracking 不强拦 |
| 横向不变 | `overflow-x:auto` 保留，pane 宽于右栏时横向滚动 |
| host 缺失兜底 | renderer 未拿到 host 时退回条件 `terminal.scrollToBottom()` |

## 十、迁移影响与风险

- **受影响**：所有复用 `SlotTerminalSurface` 的场景（需求详情页右栏 + main agent 组终端弹窗）。
- **打法**：共享层一次改，双场景同验；CSS 与 renderer 改动小、彼此独立可分别验证。
- **回滚 / 恢复**：纯前端改动，`git revert` 即可；若 Opt-1a' 滚轮仲裁体感不佳，可快速回退到 Opt-1b（两条并存）保底，不影响 host 外滚 + 保留历史主体。

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 滚轮被 xterm SmoothScrollableElement 抢、host 滚不动 | 中 | 滚不到底/历史 | capture-phase wheel 仲裁主动接管；浏览器实测 trackpad/快速滚动 |
| 滚轮仲裁缝合点体感生硬/抖动 | 中 | 体验 | 实测调参；不行则回退 Opt-1b 两条并存保底 |
| `height:auto` 复发 xterm 反向撑大父容器 | 低 | 布局抖动 | 父 `.terminalColumn` 固定高 + `min-height:0` 兜底（已核），双场景验证 |
| `scrollback:1000` < initial 历史窗口、历史被截断 | 中 | 丢部分历史 | 核对后端 initial capture 深度，必要时调大 scrollback |
| 选区/复制坐标 host 滚后错位 | 低 | 复制错行 | xterm6 用 getBoundingClientRect、随滚动更新；手动验收 |
| 移动端 `.terminalColumn height:auto` 回归 | 低 | 窄屏异常 | 列入手动验收 |
