# SU-CCB 项目总览

> 给人和 AI 快速摸清项目全貌。回来先读这一篇，再看《文档地图》和目录契约。

## 一句话

SU-CCB 是一个 CCB 协作协议与本地 Console 的开发工作区，用 TypeScript Console、Claude plugin、Codex skills 和 kernel reference 支撑多 agent 项目协作。

## 当前版本 / 方向

- 当前版本 / 方向：文档驱动架构迁移中，真相源从 `.ccb` 本体上移到人读 `docs/` 文档。
- 入口文档：`docs/06_决策记录/ADR-0037-文档驱动架构-真相源上移人读文档.md`
- 运行契约：`docs/.ccb/docs-structure-contract.yaml`
- 全量导航：`docs/00_文档地图.md`

## 核心模块

| 模块 | 路径 | 状态 | 说明 |
|------|------|------|------|
| Console Web | `apps/ccb-console/web` | 在用 | React + Vite 前端，提供项目、需求、任务、slot 与活动视图。 |
| Console Server | `apps/ccb-console/server` | 在用 | Fastify + Prisma 服务，负责 API、索引、投影、slot、事件与本地运行协调。 |
| Claude Plugin | `su-ccb-claude-plugin/` | 在用 | CCB Claude 侧技能、模板、运行时与 kernel reference 的插件实现。 |
| Codex Skills | `su-ccb-codex-skills/` | 在用 | Codex 侧技能、模板与脚本。 |
| Kernel Reference | `references/kernel/` | 在用 | 节点、状态、schema、capability 等协作协议真相源。 |
| Scripts | `scripts/` | 在用 | validator / capability policy 等生成脚本。 |
| Docs | `docs/` | 迁移中 | 文档驱动真相源、人读架构/需求/开发计划/模块/经验/决策。 |

## 技术栈

| 层 | 技术 |
|----|------|
| 工作区 | pnpm workspace，`pnpm@10.25.0` |
| 语言 | TypeScript / JavaScript，Markdown，YAML，少量 Python / shell |
| 前端 | React 19，Vite 7，React Router，Zustand，xterm.js，EasyMDE，react-markdown，Vitest |
| 后端 | Node.js，Fastify 5，Prisma 6，SQLite，Zod，tsx，Vitest |
| 数据/投影 | Prisma SQLite DB，`docs/` Markdown frontmatter，`.ccb/index` 派生缓存 |
| 插件 | `su-ccb-claude-plugin` 技能/模板/runtime，`su-ccb-codex-skills` 技能/模板 |

## 项目约定

- `references/kernel/` 是节点、状态、guard、capability 与 schema 的协议真相源。
- 文档驱动迁移后的业务真相源是人读 `docs/`，`.ccb` 保留索引、协调、events、locks、drafts、schemas 等机制件。
- Console DB 是可重建投影；冲突时以人读文档为准。
- `docs/00_文档地图.md` 和 `docs/.ccb/index/document-map.json` 是派生产物，不手工维护。
- `docs/01-06` 是当前人读知识库主入口，`99_归档/` 只放归档索引或退役位置语义。
