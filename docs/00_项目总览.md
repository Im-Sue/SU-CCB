# SU-CCB 项目总览

> 给人和 AI 快速摸清项目全貌。回来先读这一篇，再看《文档地图》和目录契约。

## 一句话

SU-CCB 是一个 AI 工程协作框架，由 **4 个独立 git 仓库**（multi-repo，submodule 嵌套）组成：根容器 SU-CCB（文档中枢）+ SU-Oriel 控制台 + Claude plugin + Codex skills，用结构化 spec、节点状态、review gate 和归档证据管理多 agent 协作。

## 当前版本 / 方向

- 当前版本：v1.2.0。重点是合并 gate 按需求隔离并收口，合法跨需求 docs/协调件/需求资产不再互相阻塞，同时保持 TOLERATE 只放行、不 stage、不删除。
- 当前形态：plugin sovereignty + multi-repo 拆分（决策见 `docs/06_决策记录/ADR-0042-multirepo-submodule-role-model.md`）。
- SU-Oriel 本版交付：通知收件箱、需求看板分组优化、归档按钮门控修复、通知消息修复。
- 文档驱动：真相源在人读 `docs/`，`.ccb` 保留索引/协调/events/locks/drafts 机制件（`docs/06_决策记录/ADR-0037-文档驱动架构-真相源上移人读文档.md`）。
- 运行契约：`docs/.ccb/docs-structure-contract.yaml`
- 全量导航：`docs/00_文档地图.md`（indexer 派生，始终最新）

## 仓库构成（multi-repo）

| 仓库 | 路径 | 说明 |
|------|------|------|
| `Im-Sue/SU-CCB`（根容器） | `./` | `docs/` 人读文档 + `docs/.ccb/` 工作区 + 框架脚本 + submodule 版本绑定 |
| `Im-Sue/SU-Oriel` | `su-oriel/`（submodule） | 可选控制台：`web`（React+Vite）+ `server`（Fastify+Prisma+SQLite），从本地文件/DB 投影 |
| `Im-Sue/su-ccb-claude-plugin` | `su-ccb-claude-plugin/`（submodule） | 协议内核真相源 `references/kernel/` + Claude skills + schema generators + 模板 `templates/docs/` |
| `Im-Sue/su-ccb-codex-skills` | `su-ccb-codex-skills/`（submodule） | Codex 侧 execute/consult/doc skills 与辅助脚本 |

> SU-CCB 是根容器，另外三仓为 submodule（钉定版本）。整套开发者 `git clone --recursive`；使用者单独 clone 子仓 / fork。

## 技术栈

| 层 | 技术 |
|----|------|
| 语言 | TypeScript / JavaScript，Markdown，YAML，少量 Python / shell |
| 前端（SU-Oriel/web） | React 19，Vite 7，React Router，Zustand，xterm.js，EasyMDE，react-markdown，Vitest |
| 后端（SU-Oriel/server） | Node.js，Fastify 5，Prisma 6，SQLite，Zod，tsx，Vitest；pnpm workspace 在 `su-oriel/` 内部 |
| 数据/投影 | Prisma SQLite DB，`docs/` Markdown frontmatter，`docs/.ccb/index` 派生缓存（不跟踪） |
| 内核 | `su-ccb-claude-plugin/references/kernel/`：节点 `.node.md`、capability、lifecycle、schema、lint 工具 |

## 关键命令

| 目的 | 命令 |
|------|------|
| 整套拉取（版本绑定） | `git clone --recursive git@github.com:Im-Sue/SU-CCB.git` |
| SU-Oriel 前端 / 后端开发 | `cd su-oriel && pnpm dev:web` / `pnpm dev:server` |
| SU-Oriel 构建 / 测试 | `cd su-oriel && pnpm build` / `pnpm test` |
| SU-Oriel server typecheck | `cd su-oriel && pnpm --filter su-oriel-server typecheck` |
| schema/policy dev 刷新 | `cd su-oriel/server && pnpm generate:all`（从平级 plugin 刷新已提交产物） |
| 协议内核 lint | `cd su-ccb-claude-plugin && python3 references/kernel/tools/lint_all.py` |
| 跨仓一致性检查 | `bash scripts/check-cross-repo.sh`（四仓凑齐时跑，不进单仓 CI） |

## 项目约定

- **协议真相源**在 `su-ccb-claude-plugin/references/kernel/`（plugin sovereignty）；节点已是 `.node.md`。
- **业务真相源**是人读 `docs/`；`.ccb` 保留索引、协调、events、locks、drafts、schemas 等机制件。
- SU-Oriel DB 是可重建投影；冲突时以人读文档为准。控制台只投影/触发，不写业务真相。
- `docs/00_文档地图.md` 与 `docs/.ccb/index/` 是派生产物，不手工维护、不跟踪。
- **模板真相源**在 `su-ccb-claude-plugin/templates/docs/`（plugin 生成读自己那份）。
- `docs/01-06` 是人读知识库主入口，`99_归档/` 只放归档索引或退役位置语义。
